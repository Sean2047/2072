# 概念层词条切分方法（v1，2026-07-16，第1部分高频概念34条批量产出）

**状态**：概念层批量首轮（词典"一、方法论与总框架"至"四、时间结构与滞后量"共34条：方法论8＋核心变量V1–V8共8＋主因果链与核心机制10＋时间结构与滞后量8）已通过 check_entries.py 全部校验，本文件固化本轮确立的做法，供后续概念层批量（词典"五、路径与地理空间""六、制度经济与治理""七、技术与社会心理"，约41条）直接套用。原型3条（bsc/供应票据体系/裸露期）确立的基本格式在本轮延续，本文件补充批量化时的取舍规则。

## 0　概念词条的内容边界（D-046 既定，本轮延续）

概念词条正文 = 词典定义原文（原样引用，不改写）+ "机制展开"指针列表（按主题指向已切分的章节/索引词条，不重复机制推导）。概念词条不承担机制论证——论证已经在对应的章节词条里做过一次，概念词条只做"这个术语是什么、去哪里看它的展开"的定位功能。这是概念词条与章节词条唯一的内容边界，本轮严格遵守，无一例外。

## 1　字段口径

| 字段 | 做法 | 理由 |
|---|---|---|
| `type` / `layer` | 固定 `概念` / `理论` | 词典术语均为机制概念，理论层承载 |
| `source` | `v156 §X.X 定位段 + 词典条目`；若术语定义在前言/凡例而非编号节内，写`v156 前言XX段` | 与写作规范[V]项可验证性要求对齐；概念词条的source比章节词条更依赖多点定位（词典条目本身+正文首现/主要展开段），允许"+"连接多个锚点 |
| `paths` | 本轮34条全部为`全路径`（第1–2部分方法论/变量/机制/时间框架术语，均不因具体路径分叉） | 路径特定的概念（如供应票据体系=A2专属）留到后续批量按实际内容判断，不预设 |
| `domain` | 直接复用该术语主要定义所在章节词条的domain，不新开域 | 概念词条与其"主要展开节"共享同一分析域，避免域碎片化；34条全部落在既有五个域（总纲/方法论、总纲/社会模型、总纲/核心机制、总纲/滞后量框架、社会/精神健康） |
| `variables` | 若术语本身即为V1–V8之一，标该变量自身；若术语是某条主因果链/机制且链定义中明确点名驱动变量，按链定义标注（≤3个，从严）；纯方法论术语（如条件推演、统计理性）不标 | 与D-051②口径一致：标注需可回答"移除该变量则论证不成立"，概念词条同样适用 |
| `aliases` | 词典术语本身 + 常见简称（如V1条目同时挂"V1"与"自动化替代率"、L1条目同时挂"L1"与"L1技术部署"） | 供构建时术语自动链接匹配多种正文写法 |
| `related` | 至少包含：①主要展开章节词条（1–4个）；②词义邻近的其他概念词条（如V1↔劳动失位↔后劳动社会） | 概念词条之间形成互链网络是词典承接为概念词条的核心价值——原词典靠"出现章节"列表做的事，现在靠related图谱做 |
| `summary` | 定义压缩（"是什么"），优先直接压缩词典释义句，不引入词典之外的判断 | 写作规范7节：概念词条summary重心=定义 |
| `overview` | 定义+首要机制角色+边界，80–160字；素材来自该术语"主要展开节"的章节词条overview二次压缩，不重新回读v156原文重复推导 | 复用已切分章节词条的overview作为二次压缩源，效率更高且保证与章节词条口径一致；仍需满足压缩≠抽象（可追溯回章节词条本身） |

## 2　与批量效率相关的做法（本轮新增经验）

1. **复用已切分章节词条作为source二次压缩的素材源**：34条概念词条对应的原文（1.1–1.7、2.1–2.6）均已切分为章节词条，因此概念词条的overview直接以对应章节词条的summary/overview为基础做进一步压缩，而不是重新回读v156原文——两级压缩的终点仍是同一段原文，保真度不受影响，且大幅降低批量写作成本。此做法仅适用于"概念对应的正文已切分"的情况；若对应正文尚未切分（词典五/六/七部分对应4.x/5.x/6.x/7.x中尚有缺口的章节），仍需回读原文。
2. **词典原表格式不规整**（v156 5745行表格因Markdown渲染问题多行挤压为一行），批量提取术语定义时需先用章节锚点（"出现章节"列）与正文交叉核对，不能只信表格文本本身的分段。
3. **ID命名**：中文术语用全拼小写连字符（如`duotai-shehui`）；变量与滞后层级类术语直接用其代号小写（`v1`–`v8`、`l1`–`l4`），比全拼更符合正文引用习惯且更短。
