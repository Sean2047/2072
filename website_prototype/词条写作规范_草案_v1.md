# 词条摘要/概述写作规范（草案 v1）

**状态**：D-051③ 裁定材料（2026-07-15 起草）。适用于全部词条的 `summary`（一句话摘要）与 `overview`（一段概述）字段——渐进披露的第一、二层，可读性的实际承重层。原型 9 条为参照样本。

## 0　总则

摘要与概述是**压缩，不是改写**（D-045）：从原文提取并浓缩，不得引入原文没有的论断、数字、因果关系或价值判断。第三层（原文）零改动，由构建按节号提取。

## 1　语域

1. 分析语域的简明形态：陈述句、判断句为主，与源文档同一语域，只是密度更低。
2. 禁止：文学化修辞、悬念句式（"……会发生什么？"）、营销语气（"深刻揭示""令人惊讶"）、叙事化开场（"想象一下"）。
3. 禁止自指：不以"本节""本词条""这一部分"开头——词条会在悬停预览、索引卡片等脱离上下文的场合被读到。
4. 机制归因优先于描述性标签（与正文同一纪律）：摘要若给结论，须是原文可追溯的机制命题，不是概念简写。

## 2　summary（一句话摘要）

1. 长度 ≤ 45 字，单句，可用一个破折号或冒号分层。
2. 内容 = 该词条**最核心的机制论断**，不是主题描述。
   - 差："本词条介绍A1体系的社会保障结构。"（主题描述）
   - 好："A1的社会保障不是重新设计的，而是在工业时代遗留框架上叠加改造成的四层结构。"（机制论断）
3. 概念词条的 summary = 定义压缩（"是什么"），章节词条的 summary = 论点压缩（"为什么/如何"）。
4. 索引词条的 summary = 本节覆盖范围一句话（自动生成场合亦遵守）。

## 3　overview（一段概述）

1. 长度 80—160 字，单段，不分点。
2. 结构建议：核心机制一句 → 关键展开/阶段/数字一至两句 → 边界或张力一句。
3. 数字、区间、年份**照抄原文**，不四舍五入、不改写单位；原文给区间的不得只取一端。
4. 原文的斜体结论段（coda）是 overview 的优先素材——那已是作者自己的压缩。
5. 术语首现用全称（如"基本支持积分（BSC）"），概念链接由构建自动生成，正文中不手写链接。
6. A1/A2 对照内容遵守 A1 先于 A2 的文档约定。
7. 不做跨词条综合：overview 只压缩本词条对应原文，即使写作者知道其他节有相关内容（跨词条关系交给 related 字段与正文链接）。

## 4　质量控制流程

1. frontmatter 增加 `reviewed` 字段（初值 no）：AI 生成 → 作者复核语域与论断保真 → 改 yes。check_entries 可统计复核进度。
2. 原文哈希漂移告警（check_entries [D] 项）触发后：先复核 summary/overview 是否仍成立，再 `--update-lock`。
3. 抽查基线：每批切分随机抽 10% 由作者盲读 summary，若能从 summary 猜错原文立场（引入了原文没有的倾向），该批整体返工。

## 5　各词条类型的差异要点

| 类型 | summary 重心 | overview 重心 | 特殊约束 |
|---|---|---|---|
| 概念 | 定义（是什么） | 定义+首要机制角色+边界 | 正文=词典定义原文+机制展开映射，不重复推导 |
| 章节 | 核心机制论断 | 机制+关键数字+张力 | 正文=构建提取，零改写 |
| 事件 | 该示例演练了什么机制 | 时间线+设置变量+可替换性 | 接口声明字段齐全才算完成 |
| 索引 | 覆盖范围 | 各子词条一句话串联 | 构建自动生成，规范约束生成模板 |
