# 事件层词条切分方法（v1，2026-07-16，7.5.4台海试点产出）

**状态**：事件层方法试点（7.5.4台海冲突，单一事件词条）已通过 check_entries.py 全部校验（0错误0警告）。台海是重整目标点名的"最紧绷关节"（T4/D-037），选它做试点是因为它是"隔离不确定事件"设计的真正考验——事件层存在的目的就是让台海这类高度不确定的具体历史推演，不污染税制重组博弈（5.1.1）这类逻辑推演事件，也不污染理论层的机制命题本身。

## 1　接口声明四字段落地为 frontmatter

第3节模板（原为文档头手写文本）在词条化后拆成四个独立 frontmatter 字段，字段名须为英文（parse_entry 的 key 正则只认 `[A-Za-z_]+`）：

| 模板字段 | frontmatter key | 类型 |
|---|---|---|
| 时间位置 | `time_span` | 单行字符串 |
| 设置变量 | 复用已有 `variables` 字段（V1–V8），不新开字段 | 列表 |
| 演练机制 | `mechanism_tested` | 块标量（\|），prose，可含"X.X节"引用 |
| 可替换性 | `substitutability` | 块标量 |
| 反向义务 | `counter_obligation` | 块标量，理论层核销后写明核销依据（决策号+日期） |

`check_entries.py` 新增 `EVENT_REQUIRED` 校验：`type: 事件` 的词条若缺这四个字段之一，报 `[S]` 错误。

## 2　"真实超链接"的构建时实现

D-046③承诺"演练机制"字段的节号引用要变成真实超链接、构建时自动校验可达性。原 `build.py` 的 `link_refs`/`link_terms` 只处理 `extract` 提取的正文（`bodies` 字典），不触及 frontmatter 字段。本次试点在 `build.py` 补了一段：对 `type: 事件` 的词条，`mechanism_tested`/`substitutability`/`counter_obligation` 三个字段单独跑一遍 `md_to_html` → `link_refs` → `link_terms`，结果存入 `ifaces` 字典，`entry_page()` 新增 `iface` 参数消费这个结果，渲染在词条页新增的"接口声明"区块（`<dl>` 时间位置/演练机制/可替换性/反向义务）。这样"7.1.6节"这类引用即便写在 frontmatter 里也会被解析成真实链接（目标词条未切分时按 `[未收录]` 样式显示，理论层切到7.1.6后自动补链，不需要回来改事件词条）。

**已知缺口**：`check_entries.py` 的 `[R]` 引用分级审计目前只扫描 `extract` 正文，不扫描接口声明四字段——`mechanism_tested` 里的"7.1.6节"/"1.2.3节"不会被 `[R]` 统计进真悬空/待切分名单。本次未阻塞（build.py 的链接解析已独立跑通，risk 是构建产物与校验报告不完全同步），登记为小待办，理论层切到 7.1.6/1.2.x 附近时顺手把 `[R]` 审计范围扩到事件词条的接口字段。

## 3　domain 新大类：地缘

事件层内容以具体地缘/历史推演为主，不套用经济类目。新增 `地缘` 大类，本次注册 `地缘/国际冲突`（覆盖7.5全节：俄罗斯/阿以/核扩散/台海）。domain清单.md 的 `load_domain_whitelist` 正则同步加了"地缘"前缀支持。

## 4　paths 打标：事件可以跨路径

理论/技术词条的 `paths` 多数收敛到单一路径或"全路径"；事件词条如果情节本身横跨多个路径行为体的决策互动（台海事件同时呈现美国内政与北京决策两条线），`paths` 据实标注多个（本例："A1, A2"），不必强行归一。

## 5　反向义务字段与理论层核销的挂钩方式

`counter_obligation` 字段里显式写出对应的理论层决策号（如"D-037，2026-07-14核销"）和工作文件登记项编号，使反向义务的"已核销"状态可以从词条正文直接查证，不需要跳回决策日志才能确认。这是接缝规则1（"事件层每个示例声明反向义务，理论层必须消化后才能核销"）在词条层的落地方式。
