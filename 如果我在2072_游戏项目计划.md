# 《如果我在2072》游戏项目计划

**类型：** 互动推演小游戏（娱乐向）  
**平台：** 手机APP（MVP先做微信H5）  
**目标：** 用户输入所在地点和个人信息，先推演该地点2072年的图景，再把用户放进去，生成个性化日常生活画像。真实感 × 娱乐性。

---

## 一、推演方法总架构（核心）

本游戏采用**两阶段推演**，区别于"选大区→看通用描述"的简单方式：

```
第一阶段：地点推演
────────────────────────────────────────────
用户输入城市/地区
    ↓
提取该地点的现实数据档案（经济类型、城市化程度、产业结构…）
    ↓
应用《2072》世界观逻辑（自动化冲击 × 能源转型 × 社会路径）
    ↓
生成"2072年这个地方的样子"（城市命运图景）
────────────────────────────────────────────

第二阶段：个人定位
────────────────────────────────────────────
用户输入个人信息（年龄、职业类型、经济状况、价值观）
    ↓
在该城市的2072年社会结构中，分配社会角色
    ↓
生成"你在这个地方的日常生活画像"
────────────────────────────────────────────
```

**这个设计的核心价值：** 同一个人选不同城市，结果完全不同；同一城市里不同类型的人，命运也完全不同。组合空间大，重复可玩性高，分享时有"我和朋友结果不一样"的话题效应。

---

## 二、体验流程

```
引导页（5秒氛围建立）
    ↓
输入城市（文字搜索 / 选择，这是第一个也是最重要的输入）
    ↓
城市推演动画：「正在分析 [城市名] 的2072年…」（3秒）
    ↓
揭晓城市命运（一段城市图景先给出来，制造悬念）
    ↓
个人问卷（8-10题，刷卡式，约90秒）
    ↓
个人定位动画：「正在分配你的社会坐标…」（3秒）
    ↓
完整结果页（城市图景 + 你的角色 + 日常生活叙事 + 数据 + 标签）
    ↓
分享卡片
```

**关键体验节点：** 先看到城市命运，再看到自己在里面的位置——这个顺序制造"先建世界，再放人进去"的代入感，比直接出结果更有叙事张力。

## 三、城市档案系统（地点推演基础）

### 城市档案结构

每个城市/地区对应一份档案，包含：

| 字段 | 说明 | 示例（成都） |
|------|------|------|
| 经济类型 | 主导产业类型 | 新能源+文创+服务业 |
| 城市化等级 | 一线/新一线/二线/三线/农村 | 新一线 |
| 产业替代风险 | 主导产业的AI替代程度 | 中偏低 |
| 人口趋势（当前） | 流入/稳定/流出 | 持续流入 |
| 治理模式 | 决定社会路径 | 中国路径 |
| 特殊标记 | 影响2072年命运的特殊因素 | 盆地气候宜居，文创优势 |

### 地点匹配策略

- **一级覆盖（精准档案）：** 预建50-60个城市档案，覆盖中国主要城市 + 全球主要都市
- **二级匹配（近似推演）：** 未收录城市，按"省份/国家+城市规模+经济类型"匹配最近似城市
- **文字提示：** "你的城市没有专属档案，我们按照与[XX城市]相似的情况推演"

### 核心城市2072年命运预设（中国）

以下是城市档案的推演结果方向——这是内容设计的核心工作：

**上海**  
金融业被AI深度渗透，功能转型为全球AI治理协调中心之一。顶层（AI治理精英、跨国机构）与底层（基础分配依赖的服务业留存人口）之间中产层空洞化完成。浦东部分区块成为"自动化展示区"，外来务工人口大幅回流内地。

**深圳**  
中国最早完成制造业机器人替代的城市。2072年深圳已高度极化：南山/福田仍是技术精英聚集地；原关外大量社区成为高密度基础收入区，居住密度高，流动性低。

**成都**  
转型相对成功。新能源基建完善，文创+意义经济发达，气候宜居，承接了大量"主动降速"的内迁人口。自持社区模式在成都发育最好，是中国路径里相对有质感的一类。

**东莞/佛山**  
制造业被机器人完全接管，工厂城市历史终结。大量人口已在2030年代完成迁移，留下来的是老龄人口和无法迁移的群体。2072年作为"遗留型城市"存在——有齐全的基础服务，但缺乏新的社会活力。

**鹤岗/玉门（资源型衰退城市代表）**  
煤炭需求在2040年代因聚变落地而崩溃，城市进入深度收缩。2072年是中国最典型的"遗忘地带"——政府维持基础服务，但几乎没有经济活动，大量空置建筑，留守老人为主。

**拉萨/乌鲁木齐（西部特殊路径）**  
基础设施覆盖完整（中国路径的均一化特征），但经济自主性弱，高度依赖中央转移支付和"文化旅游"政策保护。人类劳动在旅游和传统手工业里被作为"文化资产"保留。

**北京**  
政治中枢功能强化，AI治理的官僚体系在北京高度聚集。高度稳定，高度保守。体制内的AI监督岗是最稳定的职业形态。生活有保障，但流动性极低，外地人进入门槛更高。

**全球城市（部分）**

| 城市 | 2072年命运摘要 |
|------|--------------|
| 东京 | 人口持续老龄化，机器人普及率全球最高，"空城感"强，老人与机器人共居 |
| 首尔 | 技术先行，但出生率危机+AI替代叠加，年轻人口极少，社会创新活力枯竭 |
| 纽约 | 美国路径极化典范，曼哈顿更贵更空洞，布鲁克林-皇后区分化严重 |
| 伦敦 | 欧洲民主路径，基础收入+意义政策，稳定但停滞，脱欧代价在这50年持续显现 |
| 孟买 | 遗忘地带边缘，内部极化，精英层融入全球科技体系，贫民窟数字化改造但未根本改变 |
| 内罗毕 | 非洲科技节点，跳过了工业化直接进入AI时代，但基础设施薄弱，不稳定 |

## 四、问卷设计

问题放在城市图景揭晓之后，聚焦个人定位。风格轻松，不调研感。

### 模块1：基础定位（2题）

**Q1：你今年多大？**  
滑块（确定2072年年龄 = 当前年龄 + 46）

**Q2：你是？**  
男 / 女 / 其他（影响部分叙事细节）

---

### 模块2：职业与适应（3题）

**Q3：你工作最核心靠的是什么？**  
- 🏗 身体——体力、操作、技能
- 🧠 大脑——分析、判断、专业知识
- 🤝 关系——沟通、教育、照顾他人
- 🎨 创造——设计、写作、艺术、发明
- 📚 我是学生 / 还没开始工作

**Q4：AI快速发展，你现在的状态是？**  
- 我已经在用了，挺上手的
- 有点焦虑，但还没认真学
- 觉得和我关系不大
- 我刻意不用，有我的理由

**Q5：你觉得到2072年你最难被替代的是？**  
- 我有深度的专业积累
- 我和人打交道的方式是独特的
- 坦白说，我不确定
- 我不想被替代——我宁愿让AI替掉我

---

### 模块3：经济（1题）

**Q6：你家的财务状况大概是？**  
- 资产比较多，有房有钱有余裕
- 还可以，有些存款，不太担心
- 刚好够用，没什么余裕
- 比较紧张，主要靠收入

---

### 模块4：价值观（3题）

**Q7：如果2072年不需要工作也能活，你会？**  
- 全力投入某件事（创作/科研/某个使命）
- 慢慢生活，平静就好
- 反而会焦虑，不知道干什么
- 先玩个够再说

**Q8："有意义的生活"对你来说是？**  
- 留下点什么（作品/孩子/影响力）
- 享受过程，想太多没用
- 参与某件比自己大的事
- 搞清楚自己到底是谁

**Q9：你更能接受哪种孤独？**  
- 身边有人，但没人真正懂我
- 独处，偶尔联系几个人
- 我不要孤独，我需要热闹
- 孤独？我现在就孤独

## 五、推演逻辑：从地点+个人到社会角色

### 第一步：城市命运分类

根据城市档案，每个城市在2072年落入以下某个类型（可叠加）：

| 城市类型 | 代表 | 社会结构特征 |
|---------|------|------------|
| **技术极化城市** | 上海、深圳、北京 | 精英层+基础收入层，中间空洞，流动性低 |
| **宜居转型城市** | 成都、杭州 | 自持社区发育好，意义经济活跃，相对均质 |
| **遗留型城市** | 东莞、鹤岗 | 老龄化+基础收入依赖，活力低，稳定但空洞 |
| **全球枢纽** | 纽约、伦敦、东京 | 按各自路径极化或停滞，顶层仍有全球流动性 |
| **遗忘地带** | 内陆农村、衰退资源城市 | 低度自治，传统生活方式保留，技术覆盖薄 |

### 第二步：个人的城市内定位

在确定城市类型后，用个人变量确定这个人在该城市中的层位：

**职业替代指数**（Q3 × Q4）→ 高/中/低风险

| 职业类型 | 主动拥抱 | 中性 | 回避 |
|---------|---------|------|------|
| 体力工作 | 机器人协调员（中等稳定） | 转型期遗留岗 | 高替代，基础收入依赖 |
| 脑力分析 | AI协作专家 | AI监督岗 | 高替代 |
| 人际关系 | 增强型服务 | 半人半AI混合 | 有价值但报酬下降 |
| 创造性 | 人机共创工作室 | 意义经济 | 人类手作溢价小众市场 |

**经济韧性**（Q6 × 城市类型）→ 宽裕 / 基本保障 / 脆弱

- 技术极化城市 × 资产多 = 可能是精英层，也可能只是"活得下去的老资产"
- 遗留型城市 × 资产少 = 基础收入层，生活稳定但缺乏出口

**意义感类型**（Q7+Q8+Q9）→ 内驱 / 社会认同 / 享乐 / 漂移

**2072年年龄**（Q1+46）→ 影响视角和处境

- < 30：AI原住民，没有"以前"的对比
- 30-50：转型亲历者，职业被打断过，正在重建
- 50-70：被迫转型的一代，最脆弱
- > 70：延寿受益者，身体完好但"无用感"强烈

### 第三步：组合出具体角色

以上变量组合后，输出一个具体的**社会角色**（职业形态 + 社区类型 + 收入来源），作为叙事写作的基础。

示例组合：

| 城市 | 年龄 | 职业 | 经济 | 意义感 | → 分配角色 |
|------|------|------|------|--------|-----------|
| 成都 | 35→81岁 | 脑力+拥抱AI | 中等 | 内驱 | 自持社区的AI伦理协调员，每周工作20小时，有大量时间做自己的事 |
| 深圳 | 28→74岁 | 体力+回避AI | 紧张 | 漂移 | 基础收入层居民，住在原关外的标准化公寓，没有固定工作，偶尔接临时任务 |
| 鹤岗 | 50→96岁 | 人际+中性 | 宽裕 | 社会认同 | 延寿老人，资产让他生活无忧，但整个城市活力已消失，他是"留守的富人" |
| 上海 | 22→68岁 | 创造+拥抱AI | 中等 | 享乐 | 人机共创工作室的合伙人，收入不稳定但有一定声誉，生活在浦西老弄堂里 |

## 六、结果页设计

### 三层结构

**第一层：城市2072图景（先于个人结果出现）**

3-5行文字，描述这个城市在2072年的整体面貌。第三人称，像旁白。

成都示例：
> 2072年的成都，是中国转型最平稳的城市之一。聚变电网让这里的电价接近免费，大量数字游民和"主动降速"的移民在过去二十年持续涌入。城市以自持社区为基本单元运作，每个社区有自己的农业、文化和AI协调体系。这里的人不算富，但普遍不太焦虑。

**第二层：你的个人叙事（核心）**

200-300字，第二人称，像一段小说。描述：你在哪里 → 你每天干什么 → 你的收入从哪来 → 你和周围人的关系 → 你的情感状态。

结尾通常留一个开放性张力——不是"圆满"，也不是"悲剧"，而是一个真实的困境或问题。

示例（成都，35岁→81岁，脑力+拥抱AI，内驱型）：
> 你81岁，住在成都青羊区一个叫"松下"的自持社区，这里大概有两千户人家。你的身体状况和50年前的50岁差不多——延寿项目给了你这一代人一个奇怪的礼物：更多的时间，但没人告诉你怎么用。
>
> 你的工作是社区AI协调员，每周大概工作二十小时，处理一些AI拿不准、需要人类判断的边界案例——邻里纠纷里的情感权重、某个建设方案对老人的细节影响。这个工作不算稀缺，但在松下社区你做得很好，大家认识你。
>
> 你有一个做了四十年、至今没做完的事情：研究2026-2040年代那段转型期的口述历史。你当时就在那里，但总觉得还没到下笔的时候。现在你81岁了，终于觉得：如果不写，大概就真的没人写了。

**第三层：数据 + 标签**

```
城市：       成都（宜居转型城市）
2072年年龄：  81岁
经济状态：    基本保障 ██████░░░░
职业形态：    AI伦理协调员（社区自治型）
意义感：      ●●●●○  建构中
孤独风险：    低

#自持社区居民  #软着陆成功者
#延寿困惑期   #意义感重建中
#成都路径
```

## 八、技术架构

### 数据层

城市档案以JSON格式存储，每条记录包含：
- 城市基础数据（经济类型、城市化等级、产业结构）
- 2072年推演结果（命运类型、城市图景文本）
- 标签（供前端匹配分配）

```json
{
  "city": "成都",
  "aliases": ["chengdu", "成都市"],
  "economicType": "新能源+文创+服务业",
  "urbanLevel": "新一线",
  "industryRisk": "中偏低",
  "populationTrend": "流入",
  "governancePath": "中国路径",
  "cityFate2072": "宜居转型城市",
  "cityNarrative": "2072年的成都，是中国转型最平稳的城市之一…",
  "cityTags": ["宜居转型城市", "自持社区", "意义经济活跃"]
}
```

### MVP架构（微信H5）

- **前端**：Vue 3，纯前端推演，无后端
- **城市数据**：JSON文件打包进前端，支持模糊匹配
- **推演逻辑**：规则树（城市类型 × 个人变量 → 角色索引）
- **叙事文本**：预写，按城市类型 × 角色类型索引（约80-100段）
- **分享**：Canvas生成卡片图
- **开发周期**：3-4周（含内容撰写）

### AI增强版（可选，扩展用）

- 城市档案 + 个人变量发给Claude API
- 动态生成个性化叙事（城市图景 + 个人故事各一段）
- 支持更精细的城市（无需预写所有城市的档案文本）
- 成本：约0.02-0.05元/次（可接受）

**建议：** MVP用预写文本，验证可玩性后，若用户量到位，切换AI增强版以支持长尾城市。

## 九、内容工作量估算

| 内容类型 | 数量 | 工作量估算 |
|---------|------|----------|
| 城市档案（含推演数据） | 50-60个城市 | 每个约30分钟，总计25-30小时 |
| 城市图景文本（3-5行旁白） | 50-60条 | 每条约15分钟，总计15小时 |
| 个人叙事段落（200-300字） | 80-100段 | 每段约45分钟，总计60-75小时 |
| 问卷选项打磨 | 9题×4选项 | 约4小时 |
| 标签库 | 30-40个标签 | 约2小时 |
| **合计** | — | **约100-110小时** |

**这是项目最大的工作量所在。** 叙事段落的质量直接决定游戏的口碑，不能批量生成，需要逐段打磨。

## 十、开发阶段计划

### Phase 0：城市档案 + 内容设计（4周，最高优先级）

- [ ] 确定首批50个城市清单（中国30个 + 全球20个）
- [ ] 建立城市档案结构和数据（可借助AI辅助，人工审核）
- [ ] 撰写首批城市图景文本（30个中国城市）
- [ ] 撰写首批个人叙事段落（覆盖5种城市类型 × 4种意义感类型 × 3个年龄段 = 60段）
- [ ] 设计问卷最终措辞（找5人口头测试）
- [ ] 内部可玩性测试（验证推演合理性和共鸣感）

### Phase 1：H5原型（2-3周）

- [ ] Figma原型（体验流程完整可点击）
- [ ] 城市搜索 + 匹配逻辑
- [ ] 推演规则树实现
- [ ] 基础结果页展示
- [ ] 内测10-20人，验证传播意愿

### Phase 2：APP精细化（4-6周）

- [ ] UI/UX精细化（动效、城市图景动画）
- [ ] React Native双端开发

## 十一、关键设计原则

1. **城市要有具体命运，不要模糊描述** — "深圳极化"比"中国路径城市"有力量百倍

2. **个人叙事要有张力，不要给答案** — 结尾留一个开放的困境，让用户自己想"那我会怎么做"

3. **世界观概念透过细节传出去** — "人类认证员"比"相互忽视均衡"更让人记住

4. **同城市、不同人，命运要显著不同** — 这是重复可玩和传播的核心驱动力

5. **城市图景要先于个人结果出现** — 先建世界，再把人放进去，叙事张力更强

6. **允许"有点难受"的结果** — 如果所有结果都是爽文，真实感就死了

## 十二、与《2072》项目的战略关联

这个游戏直接解决了你提到的**中观层质量差**的问题：

- 城市档案 = 地理图景的细化落地（对应世界观文件第6部分）
- 个人叙事段落 = 中观层人物处境和日常质感的系统性填充
- 用户反应 = 实时检验哪些生活路径描述有共鸣，哪些不真实
- 游戏流量 = 为后续短视频/小说内容引流的最佳入口

做这个游戏，实际上是在倒逼你完成最难写的那部分世界观内容：**具体的、有质感的、地方性的中观图景**。
