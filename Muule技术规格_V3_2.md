**技术附件：Muule 技术规格**

本附件为《2072 世界观文件》3.12 节「Muule：个人运动模块」的工程参数详情，供创作时参考。

# **一、轮组规格**

Muule 四轮形态由两类轮组构成：主轮（驱动轮）与辅助轮（从动轮）。两者轮径相同（Ø 0.60 m），高速规格一致，简化备件管理。两轮形态下辅助轮折叠收纳于车体后侧；静止变形时单自由度展开臂向下展开，触地后机械锁定，触发车体向后倾倒 45°，重心随之下移，乘员进入半躺姿态。

| 项目 | 主轮（驱动轮） | 辅助轮（从动轮） |
| --- | --- | --- |
| 轮径 | Ø 0.60 m | Ø 0.60 m（与主轮同规格） |
| 轮宽 | W 0.24 m（整轮） | W 0.12 m（标准摩托车前轮规格） |
| 数量 | 2（左右各一） | 2（左右各一，折叠收纳于车体后侧） |
| 功能 | 驱动 + 转向 + 制动（两轮/四轮均主动） | 承重 + 制动（四轮形态专用） |
| 收纳方式 | 常态展开，始终接地 | 单自由度展开臂折叠于车体后侧，静止时展开就位 |
| 展开锁定 | — | 展开臂到位后机械锁定，承重前强制锁定确认 |
| 高速转速（160 km/h） | 约 1415 rpm | 约 1415 rpm（同径同速） |
| 轮胎规格 | 高速轮胎，成熟技术区间内 | 同主轮规格 |

# **二、静止变形时序**

形态切换严格限定在车辆完全静止、乘员处于车外时执行，禁止行驶中变形。标准流程：用户下车站于车外，触发变形指令，Muule 自动完成展开（约 3 秒），用户上车关闭舱门后方可驶入行车道。进入高速路时，路缘变形区在入口前完成变形，检查区和换电区依次跟进。

| 时间 | 事件 |
| --- | --- |
| T = 0 | 车辆完全停止，驻车制动自动启动 |
| T + 0.5 s | 左右辅助轮展开臂同步解锁，开始向下展开 |
| T + 1.5 s | 辅助轮触地，车体铰链解锁，车体开始向后倾倒（约 45°） |
| T + 2.5 s | 车体到达 45° 锁定位，辅助轮展开臂机械锁定，乘员座椅自动调整至半躺姿态 |
| T + 3.0 s | 系统确认四轮形态及所有锁定状态，解除驻车制动，允许启动行驶 |
| 总时长 | 约 3 秒（静止完成，无行驶安全窗口要求） |

# **五、换电与行程管理**

取消行驶中弹射换电带设计。换电逻辑与现行高速公路加油逻辑直接对应：

① 入口换电（标准流程）：车辆在高速路入口完成变形后进入换电区，换入满电电池（短途）或大容量电池（跨集群长途，约为标准容量 2–3 倍）。大容量电池可覆盖单段高速全程，无需途中停站。

② 中途出口换电（续航不足时）：道路系统根据电池剩余电量与导航目标实时计算，电量低于安全裕量时提前提示用户驶出高速，在出口或休息站换电后再驶入。休息站换电为标准配置。提示触发阈值：预计无法到达下一出口时提前约 15 km 发出第一次提示。

③ 出口换电（收尾）：出高速后在出口换电区换回标准电池，与入口计价合并统一结算，用户无需操作。③ 出口自动换电：道路系统在出口匝道减速，车辆降至低速后退出系统控制，出口换电区换出高速行驶后的电池，与入口评估价合并计费结算。磁阻制动节点集中部署在入口加速段末端、出口减速段、紧急停车带，功能是辅助速度过渡和故障引导，不承担供能功能，节点式部署建设成本远低于全路段感应线圈。

# **六、电池规格与成本**

车载固态电池 5–7 kWh，重约 12 kg，低速模式约 50 公里续航，高速模式（160 km/h，能耗约 100 Wh/km）约 50–70 公里续航。固态电池到 2072 年约 15–25 美元/kWh（以 2025 年美元购买力为基准），每块约 75–175 美元，循环寿命约 2000–5000 次，高频使用下约半年至一年半需要更换。这个消耗性成本是换电站网络经济可行边界与物流密度直接挂钩、不会无限扩展的根本原因。

# **七、工作状态速度参数**

| 状态 | 形态 | 速度上限/区间 |
| --- | --- | --- |
| 步行跟随 | 两轮 | 20 km/h |
| 城市自动驾驶（一般道路） | 四轮 | 约 70 km/h |
| 城市自动驾驶（快速路） | 四轮 | 约 100 km/h |

两轮/四轮形态切换阈值：低于 20 km/h 场景（人行道、室内）为两轮形态；进入行车道（一般道路、快速路、高速）一律四轮形态。

# **八、停放占地参数**

Muule 空载 92 kg，无法像自行车一样手提进室内，需专用停放空间。两轮形态占地约 0.6 m×1.2 m，同等面积可停放 8–10 台 Muule，相当于停 1 辆汽车的空间——这是停放桩改造经济性的量化基础。

# **九、手机—Muule 交互与监控粒度**

步行跟随模式下 Muule 始终保持在用户身后约 2 米，位置与状态数据实时同步至城市管理系统，是配给监控体系中密度最高的行为数据来源之一（区分步行、低速骑行、高速巡航等具体行为，而不仅是位置点）。

# **十、货舱规格**

货舱为独立模块，自带两个小轮（约 Ø0.2 m）及折叠手柄。单货舱容积约 60–80 升，最大载重约 30–50 kg（在 Muule 设计载荷余量内）；前后各一接口可同时挂接两个货舱，总载货约 120–160 升。四轮货运形态下货舱与主舱流线整合，160 km/h 下的空气阻力显著低于分段式后挂拖斗构型。

# **十一、高铁载车舱规格**

车辆通道车厢宽约 3.3 米，可容纳两列 Muule 并排或单列更宽松排列，中间留通行通道。车厢供电接口功率与家用慢充相当，按高铁单程时间设计可完成一次满充。
